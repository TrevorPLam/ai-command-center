$g13: Motion tokens — shared from src/lib/motion.ts
$g14: dnd-kit sensors — shared useDndSensors() hook
$g15: HTML sanitization — all untrusted HTML via SanitizedHTML component
$g16: Security — no dangerouslySetInnerHTML outside SanitizedHTML
$g17: Offline-first — centralized IndexedDB via Dexie CommandCenterDB (src/lib/db.ts)
$g18: Storage quota — use useStorageQuota hook; warn at ≥80%

       cancelQueries → getQueryData snapshot → setQueryData optimistic

## Motion Tier Assignment (Global)
════════════════════════════════════════════════════════════
Component          | Tier   | Technique
AppShell           | Alive  | Page transitions, spring physics, layoutId
Sidebar            | Alive  | Spring width animation, layoutId active pill, accordion height
StatusBar          | Quiet  | Pulse animation (disabled if reduced motion), tooltips
RightPanel         | Alive  | Spring slide-in/out
CmdPalette         | Alive  | Spring scale-up, staggered children, backdrop fade
VoiceShell         | Alive  | Spring scale-up, waveform animation

These tiers govern ALL instances of these components in ANY module.
Modules may add module‑specific motion tiers, but must never contradict the
global assignment.

════════════════════════════════════════════════════════════
Severity | Action
S        | BLOCK — stop immediately and ask for human override; must not continue
P        | TODOC — add // TODO(PERF) comment with explanation; then continue

ST       | Informational — you may compress the rationale; no action required

## Usage in Module Specifications
════════════════════════════════════════════════════════════
Each module’s L1 registry should include a `uses` column listing only the
contracts that the task depends on. For example:
  DASH-004: uses: [$g2, $g6, $g14, $lc-dndreorder]

When an agent executes a task, it should load `_global-contracts.txt` once,
then extract only the contracts listed in `uses` for that task.
