| **C‑05** | Glass & Glow Scope | Add a `noise-overlay` utility **only for selected glass shell surfaces** (main frame, drawer surfaces, command palette, hero cards). Ordinary content cards use a quieter dark surface (`bg-[#111111]`). |
| **C‑06** | Reduced Motion Hook | Create `useMotionPreference()` that reads `prefers-reduced-motion`. When reduced motion is requested, replace spring animations with instant state changes (or very short fades). Do not simply “disable everything” – preserve clarity. |
| **C‑07** | Page Transitions | Wrap main content in `AnimatePresence` with `motion.div` keyed by `location.pathname`. Entrance: `initial={{ opacity: 0, y: 8 }} animate={{ opacity: 1, y: 0 }} exit={{ opacity: 0, y: -8 }}`. |
| **C‑08** | Focus Restoration | After closing any modal/drawer, return focus to the element that triggered it (store ref in Zustand). |
| **C‑09** | Keyboard Shortcut Hints | Use `<kbd>` tags (e.g., `⌘K`) for common actions; show on hover or in tooltips. |
| **C‑10** | Accessibility Baseline | **WCAG 2.2 AA compliance** plus robust keyboard, focus, and screen‑reader behaviour. (Do not target WCAG 3.0 as a checklist; it is not yet stable.) |

### 🎯 Motion Hierarchy (New Section)

To keep the UI **disciplined and premium**, categorise every animated element:

| Level | Use Case | Allowed Techniques | Example Components |
|-------|----------|--------------------|--------------------|
| **Alive** | Core navigation, state changes, user feedback | Spring physics (`type: "spring", stiffness: 300, damping: 30`), shared layout animations (`layoutId`), glow on hover | Sidebar expand/collapse, active nav pill, command palette stagger, chat input LED border |
| **Quiet** | Secondary elements, content reveals, tooltips | Opacity fades, very short transitions (≤150ms), no glow | Tooltips, dropdown menus, skeleton loaders |
| **Static** | Dense data tables, repeated items, non‑interactive elements | No animation – use instant changes | Long transaction lists (except loading), calendar grid cells |

**Rule of thumb:** If an interaction is **primary** (navigation, sending a message, moving a task), use spring. If it is **informational** (tooltip, hover detail), use quiet fade. If it is **data‑dense** (table rows), keep static.
